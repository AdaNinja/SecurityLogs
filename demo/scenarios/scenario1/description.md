# 钓鱼攻击场景 (Scenario 1)

## 场景描述
本场景模拟了一个典型的钓鱼攻击过程，攻击者通过发送钓鱼邮件，诱导用户点击恶意链接，从而触发一系列恶意行为。

## 攻击流程
1. 攻击者发送钓鱼邮件给目标用户
2. 用户点击邮件中的恶意链接
3. 恶意链接触发下载和执行恶意程序
4. 恶意程序在系统中建立持久化
5. 攻击者获取系统控制权

## 文件说明
- `config.json`: 场景配置文件，包含攻击参数和配置
- `attack.py`: 攻击模拟脚本
- `data/`: 存放场景相关的数据文件
  - `raw/`: 原始日志数据
  - `processed/`: 处理后的数据

## 数据收集
本场景收集以下类型的数据：
- Windows 安全日志
- Sysmon 日志
- 网络流量数据 (PCAP)
- 系统行为数据

## 标签说明
- L1: 钓鱼邮件发送和点击
- L2: 恶意程序下载和执行
- L3: 系统持久化
- L4: 攻击者控制
